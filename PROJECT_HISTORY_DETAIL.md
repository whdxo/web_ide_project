# 📜 프로젝트 상세 작업 히스토리

## 📅 2025년 12월 22일 (월)
### 🏗️ 프로젝트 구조 개편 및 개발 환경 완벽 구축 (박영선)
**1. 프로젝트 상태 정밀 진단 및 구조 파악**
- 프로젝트 전체 파일 구조(`backend`, `frontend`)를 스캔하여 현재 상태를 점검했습니다.
- `PROJECT_SUMMARY.md`와 `EDITUS_CONTEXT.md`를 분석하여 기획 의도와 현재 구현 진행률을 대조했습니다.
- 프론트엔드(`npm install`)와 백엔드(`gradlew tasks`)의 빌드 환경을 테스트하고, 의존성 설치 상태를 확인했습니다.
- 코드 내 `// TODO` 주석을 전수 조사하여 미구현 기능(로그인 로직, 에디터 API, 채팅 API 등)을 식별하고 우선순위를 파악했습니다.

**2. Git 브랜치 전략 이행 및 안전한 통합**
- 팀 가이드라인에 따라 `frontend-integration` 브랜치의 최신 코드를 로컬 작업 브랜치(`frontend/youngsun`)로 안전하게 병합(Merge)했습니다.
- 작업 중이던 코드 유실 방지를 위해 `git stash`를 활용하여 임시 저장 및 복구 절차를 수행했습니다.
- 백엔드 팀의 최신 작업 내용(`develop` 브랜치)까지 로컬로 가져와서, 프론트엔드 단독 개발이 아닌 풀스택 통합 테스트 환경을 마련했습니다.
- 불필요한 충돌을 방지하기 위해 로컬에서 선제적으로 병합을 수행하고, "Already up to date" 상태를 확인하여 PR 준비를 마쳤습니다.

**3. 로컬 개발 환경 트러블슈팅 (Backend & DB)**
- **Java 버전 불일치 해결:**
  - 프로젝트 요구사항은 Java 17이나 로컬에 Java 21이 설치되어 있어 빌드 실패가 발생했습니다.
  - `winget`을 사용하여 Java 17 (Eclipse Adoptium)을 설치하고, `JAVA_HOME` 환경변수를 재설정하여 빌드 오류를 해결했습니다.
- **데이터베이스 연결 오류 해결:**
  - 백엔드 서버 실행 시 MySQL 연결 실패(`Communications link failure`) 문제를 진단했습니다.
  - `docker-compose.yml`을 분석하여 Docker를 통해 MySQL 컨테이너를 구동시켰고, 백엔드 서버가 정상적으로 DB에 붙도록 조치했습니다.

**4. 통합 실행 테스트 및 검증**
- 백엔드(Spring Boot, 8080 포트)와 프론트엔드(Vite, 5173 포트), 그리고 데이터베이스(MySQL)를 동시에 실행하는 데 성공했습니다.
- 로컬 환경에서 실제 서버 통신이 가능한 상태임을 확인하고, 로그인 페이지 접속 등 기본 UI 동작을 검증했습니다.
