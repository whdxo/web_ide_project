

EditUs – Web IDE 프로젝트 문서 (최종 정리본)

01. 프로젝트 개요

프로젝트명: EditUs – Web IDE Platform

EditUs는 브라우저 기반으로 실행되는 웹 통합 개발 환경(Web IDE) 입니다.
사용자는 로컬 환경이나 에디터 설치 없이 웹에서 코드 작성, 편집, 저장, 채팅, 일정 관리까지 수행할 수 있습니다.

기존 IDE 대비 접근성이 높고, 교육·팀 프로젝트·협업 실습에 적합한 환경을 제공하는 것을 목표로 합니다.

⸻

02. 프로젝트 목표
    •	웹에서 실행되는 기본 IDE 기능 구현
    (파일 트리 · 코드 작성 · 저장 · 탭 시스템)
    •	팀 협업을 위한 실시간 채팅 기능 제공
    •	개발 생산성을 높이는 일정 관리 기능 제공
    •	AI를 활용한 코드 리뷰 기능 제공(선택)
    •	React + Spring Boot + MySQL + WebSocket 기반 풀스택 개발 경험
    •	AWS EC2 배포 및 운영 경험 습득

03. 팀 구성 및 역할 분담

부팀장	방영진	실시간 채팅 API / 배포 구조 정리 / 아키텍처 가이드
팀원	이세종	파일 트리 & 파일 저장 API / 저장 전략 설계 (디바운스·diff)

•	백엔드 4명 + 프론트 2명

04. 개발해야 할 핵심 기능

A. MVP (12/27까지 반드시 완성)

1. 회원 인증
   •	Spring Security + JWT 기반 로그인/회원가입
   •	JWT 발급/검증/필터 구조
   •	프론트 로그인 페이지 연동

2. 파일/폴더 트리
   •	폴더 · 파일 생성/삭제/조회 API
   •	MySQL 기반 계층형 구조
   •	프론트 파일 트리 UI 연동

3. 코드 편집기 (Monaco Editor)
   •	Monaco Editor 렌더링
   •	파일 열기/저장 기능
   •	디바운스 기반 자동 저장

4. 실시간 채팅(WebSocket)
   •	Spring WebSocket(STOMP)
   •	사용자 접속 알림
   •	프로젝트(팀) 단위 채팅방

5. IDE 기본 레이아웃
   •	에디터/파일트리/채팅 패널 구성
   •	전체 UI 구조 정리
   •	Tailwind 기반 스타일링

⸻

B. 후순위 기능(시간 여유 시)

6. 일정 관리 기능
   •	팀 Todo Board(Kanban)
   •	개인 Todo
   •	마감일·태그 관리

7. AI 코드 리뷰
   •	OpenAI API 연동
   •	코드 분석 결과 패널 제공

8. 공동 작업 감지 (CRDT X)
   •	접속자 표시
   •	현재 파일 보고 있는 사람 표시
 - 다른 사람이 화면 작업중일때 잠금

05. 개발 환경 구성

1) 프론트엔드
   •	Framework: React 기반 CSR
   •	Bundler: Vite
   •	Language: TypeScript
   •	Editor: Monaco Editor
   •	State Management: Recoil 또는 Zustand(회의 결정)
   •	Styling: TailwindCSS
   •	HTTP Client: Axios

⸻

2) 백엔드
   •	Framework: Spring Boot 3.x
   •	Language: Java 17
   •	Build Tool: Gradle
   •	Security: Spring Security + JWT
   •	DB ORM: Spring Data JPA
   •	WebSocket: STOMP 기반 실시간 통신
   •	API Test: Postman

⸻

3) 데이터베이스
   •	DB Engine: MySQL
   •	관리툴: MySQL Workbench
   •	저장 전략:
   •	MVP: 파일 내용까지 MySQL 저장
   •	확장: 파일 내용은 S3, 메타데이터는 DB (고도화)

⸻

4) 인프라/운영환경
   •	서버: AWS EC2 (Ubuntu)
   •	Reverse Proxy: Nginx
   •	HTTPS: Let’s Encrypt
   •	Docker: backend + db 컨테이너 구성
   •	배포 전략:
   •	프론트: Nginx 정적 파일 서빙
   •	백엔드: Docker 실행
   •	DB: Docker(MySQL)

⸻
